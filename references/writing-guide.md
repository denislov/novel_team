# Novel Writing Guide

This reference replaces the old internal writing bundle. It is the canonical writing and project-memory guide for Novel source files.

## Standard Artifact Chain

Maintain these files as the long-lived memory for a project:

1. `PROJECT.md`
2. `CHARACTERS.md`
3. `TIMELINE.md`
4. `ROADMAP.md`
5. `STATE.md`
6. `CHARACTER-CARD.md`
7. `CHAPTER-OUTLINE.md`
8. `CHAPTER.md`
9. `REVIEW.md`

Use the matching files under `templates/` when creating or rewriting these artifacts.

## Working Order

### New Project

1. Use `references/novel-settings-template.md` to collect the setup.
2. Create `PROJECT.md`.
3. Establish `CHARACTERS.md`, `TIMELINE.md`, `ROADMAP.md`, and `STATE.md`.

### Chapter Workflow

1. Read `PROJECT.md`, `CHARACTERS.md`, `TIMELINE.md`, and `STATE.md`.
2. Build or refresh `CHAPTER-OUTLINE.md`.
3. Write `CHAPTER.md`.
4. Sync state changes back into `STATE.md`, `TIMELINE.md`, and `CHARACTERS.md`.
5. Record review results in `REVIEW.md`.

## Core Principles

- Show, don't tell.
- Values should emerge through choices, not slogans.
- Keep characters, time, and causality consistent.
- Supporting characters need their own motives.
- Preserve tension and pacing without rushing or padding.
- Major payoffs must respect established rules.
- Respect the chapter word budget. Finish one `must_land` result per chapter, and roll overflow into the next chapter instead of inflating the current one.

## Style Rules

- Prefer verbs and concrete nouns over empty adjectives.
- Mix short and long sentences.
- Use specific sensory details when scenes need immersion.
- Keep dialogue tied to character identity.
- Avoid preachy summary lines and generic AI cadence.

## Self-Check

- Does this chapter advance plot, character, or both?
- Did this chapter complete its `must_land` without overrunning the hard ceiling?
- Do actions match each character's history, incentives, and personality?
- Are time, age, season, and continuity still aligned?
- Did we avoid deus ex machina, dumbed-down antagonists, and filler?
- Did we leave a meaningful hook or next-step signal?
- Do `STATE.md`, `TIMELINE.md`, or `CHARACTERS.md` need updates?
